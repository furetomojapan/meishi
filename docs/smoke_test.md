# スモークテスト チェックリスト

リファクタリングの各フェーズ末・デプロイ後に必ず実施する。所要約10分。
1つでも✕があれば次フェーズに進まず、ロールバック（refactoring_plan.md 付録B）を検討する。

## 準備
- 確認用ユーザー: 自分の名刺（フルURL `…#zz〜` または `?id=`）とタグ用URL（`…#zt〜`）を控えておく
- ブラウザの開発者ツール（Console）を開いた状態で行うとなお良い

## 1. 名刺閲覧（フルURL）
- [ ] カードが表示される（表面・文字・背景）
- [ ] タップでフリップし裏面（アピール）が出る。空欄アピールは表示されない
- [ ] リンク（SNS等）が開く
- [ ] シェア・QR・画像ダウンロードが動く
- [ ] 「🔒 プライバシー」リンクでモーダルが開く

## 2. 名刺閲覧（タグ用URL `zt〜`）
- [ ] 「🔒 限定表示」の注記が出る
- [ ] 電話番号・住所が「タグ用の値」または非表示になっている
- [ ] 「名刺を編集」「タグ仲間」「STORE」ボタンが表示されない

## 3. 編集（PIN認証）
- [ ] 「名刺を編集」→ PIN入力 → 正しいPINで編集パネルが開く
- [ ] わざと間違える → 「あと◯回」表示。5回で15分ロックのメッセージ
- [ ] 端末記憶（30日）が効いていれば PIN入力なしで開く
- [ ] 表示名・リンク・住所・電話・タグ用連絡先を変更 → SAVE → 再読込で反映
- [ ] FREEユーザーはリンク1件まで（PROは5件）

## 4. タグ
- [ ] タグタブに「👀 …今◯人に見えています」が出る
- [ ] 全体公開トグルON→確認ダイアログ→1枠消費される
- [ ] タグ保存→確認ダイアログ→成功メッセージ。24時間制限の表示
- [ ] 「タグ仲間」→ 一覧が出る → 他人の名刺がタグ用URL（zt〜）で開く

## 5. 管理画面
- [ ] IDなしでサイトを開く → 管理者ログイン画面
- [ ] 正しいパスワードでログイン（初期パスなら強制変更モーダル）
- [ ] ユーザー一覧が表示される
- [ ] 新規ユーザー作成 → 一覧に出る
- [ ] PRO/FREE切替・+G切替が反映される
- [ ] ユーザー編集（情報・タグ・タグ用連絡先）→ Save → 反映
- [ ] PIN表示/設定/リセットが動く
- [ ] 管理パスワード変更（8文字以上）が動く

## 6. 新規登録
- [ ] welcome.html からメール登録 → 完了メールが届く → URLから初回PIN設定ができる

## 7. 技術チェック
- [ ] ブラウザConsoleに赤いエラーが出ていない
- [ ] 通信失敗（GAS未反映等）の場合に画面が白くならない
